## 2. v0.07 总体设计

### 2.1 核心转向：不向后兼容

| 现状 (v0.06) | v0.07 |
|--------------|-------|
| `try-catch` 解析为 `Stmt::Try`，解释器独立分支 | **移除**。错误处理走 `Result<T,E>` + `?` + `match` |
| `serve as http/mcp/repl/stdio` 解析为 `Stmt::Serve` | **移除**。云服务走 `Router::new()` / `McpServer::new()` 显式 API |
| `req.body` 手动 `json.parse` | `req.json()` 一行搞定，返回 `Result<Value, ParseError>` |
| `Router::new()` 在解释器中但**无 :: 语法** | `::` 作为一等 token (`ColonColon`)，`Router::new()` 可被解析器正确识别 |
| 无 e2e 验证 | `test_data/v0.07_validation.mora` 12 项 typeck 验证 |

### 2.2 保留的独立 API

以下 **不是糖，是独立功能**，不受影响：
`route` / `observe` / `span` / `with` / `stream` / `tool` / `record_tokens`

## 4. 各 commit 详细说明

### 4.1 v0.07.0: 移除向后兼容旧语法糖

**目标**：彻底删除 try-catch 和 serve as 的底层支持，不再保留 Stmt 中间表示。

**改动**：

- `src/lexer.rs`：TokenType 枚举移除 `Try`/`Catch`/`Serve`/`Http`/`Mcp`/`Repl`/`Stdio`/`On`，降级为普通 Identifier
- `src/ast.rs`：删除 `Stmt::Try` / `Stmt::Serve` / `ServeProtocol` / `RouteDecl` / `HttpMethod` 及 impl
- `src/parser.rs`：删除 `try_statement()` / `serve_statement()` / `is_http_route_line()` / `is_mcp_tool_line()` / `parse_http_route()` / `parse_mcp_tool_route()` 共 ~180 行
- `src/interpreter.rs`：删除 `Stmt::Try` 执行分支 / `Stmt::Serve` 执行分支 / `serve_stdio_echo()` / `build_ai_error_static()` / `return_in_try_does_not_trigger_catch` 测试
- `src/typeck.rs`：删除 `Stmt::Try` typeck 分支 / `Stmt::Serve` typeck 分支（含 port 校验 / route handler 校验）
- `src/lsp/providers.rs`：删除 Try/Serve 相关的 walk_stmt / walk_expr / completion / folding 分支
- `src/http_server.rs`：移除 `HttpMethod` import，更新注释
- `src/main.rs`：banner 更新为 v0.07，移除 `catch e: AiError` / `serve` 等关键字
- `Cargo.toml`：version `0.0.6` → `0.0.7`
- `test_data/typeck_errors.mora`：类别 11 try-catch 测试改为注释废弃

**验收**：
- `cargo build` ✅
- `cargo build --bin mora-lsp` ✅
- `cargo test --lib` 47 passed ✅
- `mora --check typeck_errors.mora` 23 errors（类别 11 废弃）✅

### 4.2 v0.07.1: req.json() + :: 语法 + e2e 验证脚本 + Benchmark

**目标**：补齐最后 3 个遗留项 + 修复 `Router::new()` 的解析器支持。

**改动**：

- `src/lexer.rs`：`:` 后跟 `:` 切为 `TokenType::ColonColon` 新 token
- `src/ast.rs`：Expr 加 `NamespaceRef { namespace, name, span }` 变体
- `src/parser.rs`：
  - `primary()` 中 `IDENT + ColonColon + IDENT` 解析为 `NamespaceRef`
  - `call()` 中 `NamespaceRef + LParen` 解析为 `Call(qualified_name)`
  - `ColonColon` 后的 `.route()` 链式调用自动转为 `Call("Router::route")`
  - `consume_method_name()` 兼容 `Route` 关键字
- `src/interpreter.rs`：
  - `evaluate(NamespaceRef)` → 拼接 `"Router::new"` / `"McpServer::new"` 等 builtin
  - `call_method(Value::Dict, "json")` → `req.json()` 实现，解析 body 返回 `{ok: ...}` 或 `{err: "ParseError: ..."}`
- `src/typeck.rs`：`NamespaceRef` match arm `Type::Any`
- `src/lsp/providers.rs`：`NamespaceRef` match arm
- `test_data/v0.07_validation.mora`：12 项 e2e typeck 验证
- `examples/bench_server.mora`：Router::new() 显式 API benchmark 脚本

**验收**：
- `cargo build` ✅
- `cargo test --lib` 47 passed ✅
- `mora --check v0.07_validation.mora` 0 errors ✅
- `mora --check bench_server.mora` 0 errors ✅

## 6. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| try-catch 删除方式 | 全链路删除（lexer→ast→parser→interpreter→typeck→lsp） | 不留无用代码，静态检查强制走 Result<?> |
| serve as 删除方式 | 全链路删除 + ServeProtocol/RouteDecl/HttpMethod 类型删除 | 显式 API 已是唯一路径 |
| `req.json()` 签名 | `dict.json() → Result<Value, ParseError>` | 与 `?` 操作符兼容，空 body/非 JSON → err |
| `::` 语法 | 新增 `TokenType::ColonColon` + `Expr::NamespaceRef` | 不再需要 hack builtin 注册表 |
| Benchmark 脚本 | 写为 `.mora` 脚本配合外部工具（ab/wrk） | 压力测试由专业工具负责，Mora 负责服务逻辑 |
