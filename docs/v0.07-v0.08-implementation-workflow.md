## 2. 总体设计

### 2.1 核心转向

| 维度 | v0.06 | v0.08 |
|------|-------|-------|
| 错误处理 | `try-catch` 语句 | `Result<T,E>` + `?` + `match`（try-catch 已删除） |
| 云服务 | `serve as http/mcp` 魔法块 | `Router::new()` / `McpServer::new()` 显式 API |
| JSON 解析 | 手动 `json.parse` | `req.json()` / `str.json()` 返回 `Result<Value, ParseError>` |
| : :语法 | 无 | 一等 token `ColonColon`，支持命名空间引用 |
| 类型系统 | 扁平 20 种基础类型 | 新增 `Type::Trait(name)` + `Type::Struct(name, traits)` |
| 抽象 | 无 | `trait` + `impl ... for ...` + `dyn` dispatch |

### 2.2 保留的独立 API

以下**不是糖，是独立功能**，不受影响：
`route` / `observe` / `span` / `with` / `stream` / `tool` / `record_tokens`

## 4. 各 commit 详细说明

### 4.1 v0.07.0: 移除向后兼容旧语法糖

**目标**：彻底删除 try-catch 和 serve as 的底层支持。

**改动**：
- `src/lexer.rs`：TokenType 移除 `Try`/`Catch`/`Serve`/`Http`/`Mcp`/`Repl`/`Stdio`/`On`，降级为普通 Identifier
- `src/ast.rs`：删除 `Stmt::Try` / `Stmt::Serve` / `ServeProtocol` / `RouteDecl` / `HttpMethod` 及 impl
- `src/parser.rs`：删除 `try_statement()` / `serve_statement()` + 6 个辅助函数，共 ~180 行
- `src/interpreter.rs`：删除 `Stmt::Try` / `Stmt::Serve` 执行分支、`serve_stdio_echo()`、`build_ai_error_static()`、测试
- `src/typeck.rs`：删除 Try/Serve typeck 分支
- `src/lsp/providers.rs`：删除 Try/Serve walk/completion/folding 分支
- `src/main.rs`：banner 更新为 v0.07
- `Cargo.toml`：version 0.0.6 → 0.0.7

**验收**：cargo build ✅ | cargo test 47 passed ✅ | typeck e2e 23 errors ✅

### 4.2 v0.07.1: req.json() + :: 语法 + e2e 验证脚本 + Benchmark

**目标**：补齐最后 3 个遗留项 + 修复 `Router::new()` 的解析器支持。

**改动**：
- `src/lexer.rs`：`:` 后跟 `:` 切为 `TokenType::ColonColon`
- `src/ast.rs`：Expr 加 `NamespaceRef { namespace, name, span }`
- `src/parser.rs`：`primary()` 中 `IDENT + ColonColon + IDENT` → `NamespaceRef`；`call()` 中 `NamespaceRef + LParen` → `Call(qualified_name)`；`consume_method_name()` 兼容 `Route` 关键字
- `src/interpreter.rs`：`evaluate(NamespaceRef)` → `Router::new` / `McpServer::new` 等 builtin；`call_method(Value::Dict, "json")` → `req.json()` 实现
- `src/typeck.rs`：`NamespaceRef` → `Type::Any`
- `src/lsp/providers.rs`：`NamespaceRef` match arm
- `test_data/v0.07_validation.mora`：12 项 e2e typeck 验证
- `examples/bench_server.mora`：Router::new() benchmark 脚本

**验收**：cargo test 47 passed ✅ | v0.07_validation.mora 0 errors ✅ | bench_server.mora 0 errors ✅

### 4.5 v0.08.0: trait + interface 基础实现

**目标**：按规划书第 4 章，实现 trait 定义、impl 块、dyn 类型标注的完整语法链。

**改动**：

**AST（ast.rs）**：
- 新增 `Stmt::TraitDef { name, methods: Vec<TraitMethod>, span }`
- 新增 `Stmt::ImplDef { trait_name, for_type, methods: Vec<FnDef>, span }`
- 新增 `TraitMethod` / `FnDef` 结构体
- 新增 `Expr::DynTrait { trait_name, span }`

**Lexer（lexer.rs）**：
- 新增 `Trait` / `Impl` / `Dyn` / `Self_` 4 个关键字 token

**Parser（parser.rs）**：
- `declaration()` 新增 `Trait` / `Impl` 分支
- 新增 `parse_trait_def()` — 解析 `trait Name ... fn sigs ... end`
- 新增 `parse_impl_def()` — 解析 `impl TraitName for TypeName ... fn bodies ... end`
- `let_declaration()` 支持 `let x: dyn Trait = expr` 语法
- 总新增 ~90 行 parser 代码

**Type 系统（typeck.rs）**：
- 新增 `Type::Trait(String)` / `Type::Struct(String, Vec<String>)` 变体
- `TypeChecker` 新增 `trait_registry` / `impl_registry` 字段
- 新增 `TraitTypeDef` 内部结构体
- `compatible_with()` 扩展：Trait 同名兼容、Struct 与 Trait 兼容
- `from_hint()` 支持 `"dyn:Name"` 解析
- `check_stmt` 新增 TraitDef / ImplDef 完整性检查
- `body_has_return()` 扩展：`ImplDef` 的方法体递归检查
- 新增 `DynTrait` 的 span 辅助

**解释器（interpreter.rs）**：
- 新增 `TraitInfo` / `TraitMethodSig` 结构体
- `Interpreter` 新增 `trait_registry` / `impl_table` 字段
- `Value::TraitObject` 新变体，携带 `data: Box<Value>` + `vtable`
- `execute` 新增 TraitDef / ImplDef 处理
- `evaluate` 新增 DynTrait 求值 → TraitObject 构造
- 新增 `eval_dyn_trait()` 方法
- `value_to_json` / `type_name` / `Display` 所有 match arm 补完

**LSP（providers.rs）**：
- semantic tokens 新增关键字高亮
- completion 关键字列表新增 `trait` / `impl` / `dyn` / `Self`
- foldingRange 支持 TraitDef / ImplDef
- walk_stmt / walk_expr 新增 ImplDef 方法体递归
- span 参考新增 TraitDef / ImplDef

## 6. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| try-catch 删除 | 全链路删除（lexer→ast→parser→interpreter→typeck→lsp） | 不留无用代码 |
| serve as 删除 | 全链路删除 + 类型删除 | 显式 API 唯一路径 |
| `req.json()` 签名 | `dict.json() → Result<Value, ParseError>` | 与 `?` 兼容 |
| `::` 语法 | `TokenType::ColonColon` + `Expr::NamespaceRef` | 解析器一等支持 |
| `consume_method_name()` | 32 关键字通用映射表 | 鲁棒性，新增关键字自动可用 |
| trait 设计 | `dyn trait` 动态调度 + vtable | Rust 心智，不强制 monomorphize |
| vtable 构造 | `HashMap<String, HashMap<String, Value>>` | 简单直接，第 1-4 周够用 |
| ImplDef 方法体 | 复用 `FnDef` 结构体，递归 typeck | 与 TaskDef 同构 |
| TraitMethod vs FnDef | TraitMethod 无 body，FnDef 有 body | 语义清晰，typeck 区分检查 |
