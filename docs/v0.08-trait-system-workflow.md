## 7. 架构总结

### 7.1 Trait 系统架构

```
┌──────────────────────────────────────────────────────────────┐
│ 语法层                                                      │
│   trait Name [: Parent1, Parent2]                            │
│     fn method1(params) -> RT [= default_expr]                │
│   end                                                        │
│   impl Name for Type                                          │
│     fn method2(params) = expr | do ... end                    │
│   end                                                        │
│   let x: dyn TraitName = TraitName::new("ForType")            │
│   x.method(args)                                              │
└──────────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────────┐
│ 运行时 dispatcher                                            │
│   dispatch_trait_method(data, method, args)                  │
│     1. 读 data._type → ForType, data._trait → TraitName     │
│     2. BFS 祖先 chain: [TraitName, Parent1, Parent2, ...]    │
│     3. 对每个 tname:                                         │
│        a. 找 __impl_<tname>_<ForType>_<method> (具体 impl)  │
│        b. 找 __impl_<tname>_<tname>_<method> (默认实现)     │
│     4. call_task(impl_method, [self, ...args])               │
│        → self 自动包装为 TraitObject (body 内递归 dispatch)  │
└──────────────────────────────────────────────────────────────┘
```

### 7.2 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| dyn dispatch 路径 | `call_method` TraitObject 分支 → `dispatch_trait_method` | 不通过 vtable closure，直接按 data._type 查 env |
| impl method 注册 | `__impl_<Trait>_<ForType>_<method>` | 简单字符串拼接，env.get 即查 |
| 默认实现 | 注册为 `__impl_<Trait>_<Trait>_<method>` | self 类型 = trait 名，dispatcher fallback |
| 祖先查找 | BFS parents chain（防循环） | trait 图可能是 DAG（菱形继承等） |
| self 包装 | call_task 检测 `_trait` 字段自动包装为 TraitObject | 让 trait 默认 body 内的 `self.method()` 走 dispatch |
| 方法签名 | trait 和 impl 都用 `fn name(params)` | 统一风格，无特殊 `self` 语法 |
| 默认实现 body | `= expr` 单表达式 | 简洁直观，与 impl `= expr` 一致 |

## 9. v0.08 已知局限

| 局限 | 说明 |
|------|------|
| 无 self-less 方法 | trait method 必须含 `self` 第一个参数 |
| 默认实现只支持单表达式 | `= expr`，不支持 `do ... end` 块 |
| 无 trait 泛型 | trait 不接受泛型参数 |
| 无 where 子句 | `impl Foo for Type where Type: Bar` 不支持 |
| Monomorphize | 不支持单态化（所有 dispatch 都是 dyn） |
| 循环继承检测 | typeck 有循环防护但无 warning |

## 10. v0.08 → v0.09 衔接

按规划书第 4 章 5-8 周：**泛型系统**。v0.08 trait 系统提供的基础：

- `Type::Trait(name)` / `Type::Struct(name, traits)` — 多态类型基建
- `collect_trait_methods_recursive` — 祖先方法收集可复用
- `TraitTypeDef` / `TraitInfo` — trait 元数据完备
- dyn dispatch chain — 单态化可复用为"生成具体类型 + 方法重写"
- `Type::Result_(T, E)` — 已有泛型原型（Box<Type> 递归）
