## 2. 关键代码改动

### 2.1 `Value::TraitObject` 重构

```rust
// v0.08 (旧)
TraitObject {
    data: Box<Value>,
    vtable: HashMap<String, HashMap<String, Value>>,
}

// v0.09 (新)
TraitObject {
    for_type: String,
    trait_name: String,
    data: Box<Value>,
}
```

`vtable` 字段删除——旧版用 vtable 占位 closure，实际 dispatch 走 `dispatch_trait_method()` 通过 `__impl_*` 注册名查 env，不通过 vtable closure。`vtable` 是死代码。

### 2.2 共享函数

```rust
pub(crate) fn impl_method_key(trait_name: &str, for_type: &str, method: &str) -> String {
    format!("__impl_{}_{}_{}", trait_name, for_type, method)
}

pub(crate) fn default_impl_method_key(trait_name: &str, method: &str) -> String {
    format!("__impl_{}_{}_{}", trait_name, trait_name, method)
}

pub(crate) fn collect_required_methods<'a>(
    trait_registry: &'a HashMap<String, TraitInfo>,
    trait_name: &str,
) -> Vec<&'a str> { /* BFS 去重 */ }
```

`collect_required_methods` 给 `construct_trait_instance`（构造时完整性检查）和潜在 LSP 查询复用，与 typeck 的 `collect_trait_methods_recursive` 同构。

## 4. v0.09 → v0.10 衔接

| v0.10 需求 | v0.09 准备情况 |

| trait 泛型 `trait Foo<T>` | TraitMethod / TraitInfo / TraitTypeDef 字段命名稳定，加 `generic_params: Vec<String>` 即可 |

| `__impl_*` 加泛型部分 | `impl_method_key()` 已抽，加泛型段只改 1 处 |
| 单态化复用 dispatch chain | trait registry + dispatch_trait_method 已结构化，可生成具体类型 + 方法重写 |
| `mimo web` 集成 | MiMo Code GitHub 源码改 3 处（index.ts × 2 + web.ts × 1），rebuild mimo.exe 覆盖 npm 包 |

## 5. 已知遗留（v0.09 不修）

| 局限 | 说明 |
|------|------|
| 无 self-less 方法 | trait method 必须含 `self` 第一个参数 |
| 默认实现只支持单表达式 | `= expr`，不支持 `do ... end` 块 |
| 无 trait 泛型 | trait 不接受泛型参数 |
| 无 where 子句 | `impl Foo for Type where Type: Bar` 不支持 |
| Monomorphize | 不支持单态化（所有 dispatch 都是 dyn） |
| CLI 子命令 bug | `mimo run <file>` 路径解析（独立 bug，由 v0.09 修） |
