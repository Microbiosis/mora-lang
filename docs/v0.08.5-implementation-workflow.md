## 2. 关键代码改动

### 2.1 `Value::TraitObject` 重构（F2 主战场）

```rust
// v0.08 (旧)
TraitObject {
    data: Box<Value>,
    vtable: HashMap<String, HashMap<String, Value>>,
}

// v0.08.5 (新)
TraitObject {
    for_type: String,
    trait_name: String,
    data: Box<Value>,
}
```

`vtable` 字段删除——旧版用 vtable 占位 closure，实际 dispatch 走 `dispatch_trait_method()` 通过 `__impl_*` 注册名查 env，不通过 vtable closure。`vtable` 是死代码。

### 2.2 共享函数（F6 / F1）

```rust
pub(crate) fn impl_method_key(trait_name: &str, for_type: &str, method: &str) -> String {
    format!("__impl_{}_{}_{}", trait_name, for_type, method)
}

pub(crate) fn default_impl_method_key(trait_name: &str, method: &str) -> String {
    format!("__impl_{}_{}_{}", trait_name, trait_name, method)
}

pub(crate) fn collect_required_methods<'a>(
    trait_registry: &'a HashMap<String, TraitInfo>,
    trait_name: &str,
) -> Vec<&'a str> { /* BFS 去重 */ }
```

`collect_required_methods` 给 `construct_trait_instance`（构造时完整性检查）和潜在 LSP 查询复用，与 typeck 的 `collect_trait_methods_recursive` 同构（typeck 签名用 `Signature`，interpreter 用 `&str`，两份独立但对齐）。

### 2.3 dispatch 错误信息附 span（F4）

```rust
// v0.08.5: dispatch_trait_method 加 call_site: Span 参数
pub(crate) fn dispatch_trait_method(
    &mut self,
    for_type: &str,
    trait_name: &str,
    method: &str,
    args: Vec<Value>,
    call_site: Span,
) -> Result<Value, String>
```

运行时实际报错格式（来自 `src/interpreter.rs`）：
```
trait dispatch at line 42: receiver must be trait object or nil, got <receiver>
trait dispatch at line 42: no impl for type 'Human' method 'bar' (searched: Greeter → Greet)
trait Greeter method 'bar' has no impl for type 'Human' and no default (line 42)
```

三处分别对应 `src/interpreter.rs:1413`（非 trait object 接收者）、`src/interpreter.rs:1443`（dispatch 找不到 impl）、`src/interpreter.rs:1468`（构造期完整性检查失败）；构造错误同样附 `(line N)`，与 F4 对齐。

### 2.4 typeck 改 `Type::Trait` 注册（F3）

```rust
// v0.08.5: 删除 Type::Struct(name, Vec<String>)，统一注册为 Type::Trait
// F3 修复点：typeck 的 symbol table 不再嗅探 "_type"/"_trait" dict key
symbols.define(for_type, Type::Trait(trait_name.clone()));
```

## 4. v0.08.5 → v0.09 衔接

| v0.09 需求 | v0.08.5 准备情况 |
|------------|------------------|
| trait 泛型 `trait Foo<T>` | TraitMethod / TraitInfo / TraitTypeDef 字段命名稳定，加 `generic_params: Vec<String>` 即可 |
| impl 泛型 `impl Foo<T> for Bar` | `Type::Struct` 已删，Type::Trait 注册链路清晰 |
| `__impl_*` 加泛型部分 | `impl_method_key()` 已抽，加泛型段只改 1 处 |
| Monomorphize | 不支持单态化（所有 dispatch 都是 dyn）；v0.09 引入泛型后需重写 vtable 生成路径（不能简单复用当前 dispatch 链） |

## 5. 已知遗留（v0.08.5 不修，v0.09 / 后续处理）

| 局限 | 说明 |
|------|------|
| 无 self-less 方法 | trait method 必须含 `self` 第一个参数 |
| 默认实现只支持单表达式 | `= expr`，不支持 `do ... end` 块 |
| 无 trait 泛型 | trait 不接受泛型参数 |
| 无 where 子句 | `impl Foo for Type where Type: Bar` 不支持 |
| Monomorphize | 不支持单态化（所有 dispatch 都是 dyn） |
