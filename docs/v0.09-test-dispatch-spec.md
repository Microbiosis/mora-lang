## 1. Interpreter 改动详解

### 1.1 `Value::TraitObject` 字段扩展

```rust
/// v0.09: TraitObject 携带泛型实例化信息
///   `let c: dyn Container<number> = ...` 创建的 TraitObject
///   for_type = "BoxedNum", for_generics = []  (具体类型无泛型)
///   trait_name = "Container", trait_generics = [Type::Number]
TraitObject {
    /// 具体类型名（如 "BoxedNum"）
    pub for_type: String,
    /// v0.09 新增: for_type 的泛型参数（如 `Boxed<T>` 的 `T`）
    pub for_generics: Vec<String>,
    /// trait 名（如 "Container"）
    pub trait_name: String,
    /// v0.09 新增: trait 的泛型参数（如 `Container<number>` 的 `number`）
    pub trait_generics: Vec<String>,
    /// 实际数据（dict 等）
    pub data: Box<Value>,
},
```

**示例**：
- `let c: dyn Container<number> = Container<number>::new("BoxedNum")` →
  `TraitObject { for_type: "BoxedNum", for_generics: [], trait_name: "Container", trait_generics: ["number"], data: Dict({}) }`
- `let b: dyn Boxed<number> = Boxed<number>::new("Boxed")` →
  `TraitObject { for_type: "Boxed", for_generics: [], trait_name: "Boxed", trait_generics: ["number"], data: Dict({}) }`

### 1.2 `impl_method_key` 扩展

```rust
/// v0.09: 注册 impl method 时用的 key（含泛型签名）
/// 格式: __impl_<Trait>_<TraitGen>_<ForType>_<ForGen>_<method>
///   TraitGen / ForGen 用类型名（如 "Number" / "String"），简化版（v0.09 不含 typeck 类型）
///
/// 重要: 同一 trait 不同实例化产生不同 key，避免冲突
///   Container<number> vs Container<string> → 不同 key
pub(crate) fn impl_method_key(
    trait_name: &str,
    trait_generics: &[String],  // ["number"] 或 []
    for_type: &str,
    for_generics: &[String],   // ["T"] 或 []
    method: &str,
) -> String {
    let tg = trait_generics.join(",");
    let fg = for_generics.join(",");
    format!("__impl_{}_{}_{}_{}_{}", trait_name, tg, for_type, fg, method)
}

/// 默认实现的 key（self 类型 = trait 名）
pub(crate) fn default_impl_method_key(
    trait_name: &str,
    trait_generics: &[String],  // v0.09 新增
    method: &str,
) -> String {
    let tg = trait_generics.join(",");
    format!("__impl_{}_{}_{}", trait_name, tg, method)
}
```

**示例**：
- `impl Container<number> for BoxedNum` 的 `get` 方法 →
  `__impl_Container_number_BoxedNum__get`
- `impl<T> Container<T> for Wrapper<T>` 的 `get` 方法（实例化为 `T=number`）→
  `__impl_Container_number_Wrapper_number__get`
- `trait Container<T>` 默认 `get`（实例化为 `T=number`）→
  `__impl_Container_number__get`

### 1.4 `execute(Stmt::TraitDef)` 改动

```rust
Stmt::TraitDef { name, generics, parents, methods, span: _ } => {
    let method_sigs: Vec<TraitMethodSig> = methods.iter().map(|m| TraitMethodSig {
        name: m.name.clone(),
        params: m.params.clone(),
        return_type: m.return_type.clone(),
        has_self: m.params.first().map(|(n, _)| n == "self").unwrap_or(false),
    }).collect();
    self.trait_registry.insert(name.clone(), TraitInfo {
        name: name.clone(),
        generics: generics.iter().map(|g| g.name.clone()).collect(),  // v0.09
        parents: parents.clone(),
        methods: method_sigs,
    });
    // 默认实现：注册到 __impl_<Trait>_<TraitGen>_<method>（self 类型 = trait 名 + 泛型）
    for m in methods {
        if !m.body.is_empty() {
            // v0.09: trait_generics 从当前 TraitDef 取
            let tg: Vec<String> = generics.iter().map(|g| g.name.clone()).collect();
            let default_method_name = default_impl_method_key(name, &tg, &m.name);
            let td = Stmt::TaskDef {
                name: default_method_name.clone(),
                params: m.params.clone(),
                return_type: m.return_type.clone(),
                body: m.body.clone(),
                exported: false,
                span: m.span,
            };
            self.execute(&td)?;
        }
    }
    Ok(FlowSignal::None)
```

### 1.7 `construct_trait_instance` 扩展

```rust
/// v0.09: 构造 trait instance（Trait::new("ForType") 或 Trait<...>::new("ForType")）
fn construct_trait_instance(
    &mut self,
    trait_name: &str,
    trait_generics: &[String],  // v0.09 新增
    for_type: &str,
    for_generics: &[String],   // v0.09 新增
    call_site: Span,
) -> Result<Value, String> {
    // v0.09: 用共享 collect_required_methods 走整个 trait 继承链
    let method_names = collect_required_methods(&self.trait_registry, trait_name);
    let env = self.environment.lock().unwrap();
    for m in method_names {
        // v0.09: key 包含泛型
        let impl_name = impl_method_key(trait_name, trait_generics, for_type, for_generics, m);
        let default_name = default_impl_method_key(trait_name, trait_generics, m);
        let has_specific = env.get(&impl_name).is_some();
        let has_default = env.get(&default_name).is_some();
        if !has_specific && !has_default {
            drop(env);
            return Err(format!(
                "trait {} method '{}' has no impl for type '{}' and no default (line {})",
                trait_name, m, for_type, call_site.line
            ));
        }
    }
    drop(env);
    Ok(Value::TraitObject {
        for_type: for_type.to_string(),
        for_generics: for_generics.to_vec(),  // v0.09
        trait_name: trait_name.to_string(),
        trait_generics: trait_generics.to_vec(),  // v0.09
        data: Box::new(Value::Dict(HashMap::new())),
    })
}
```

### 1.8 `dispatch_trait_method` 扩展

```rust
/// v0.09: 用 TraitObject 的 4 个字段找具体实例化方法
fn dispatch_trait_method(
    &mut self,
    receiver: &Value,
    method: &str,
    args: Vec<Value>,
    call_site: Span,
) -> Result<Value, String> {
    // v0.09: 从 TraitObject 读 4 个字段
    let (for_type, for_generics, trait_name, trait_generics) = match receiver {
        Value::TraitObject {
            for_type, for_generics, trait_name, trait_generics, ..
        } => (
            for_type.clone(),
            for_generics.clone(),
            trait_name.clone(),
            trait_generics.clone(),
        ),
        Value::Nil => return Ok(Value::Nil),
        _ => return Err(format!(
            "trait dispatch at line {}: receiver must be trait object or nil, got {:?}",
            call_site.line, receiver
        )),
    };
    // v0.09: 沿 parents chain BFS 找所有候选 trait
    let search_chain = collect_parent_traits(&self.trait_registry, &trait_name);
    for tname in &search_chain {
        let tname_str: &str = tname;
        // v0.09: 查 trait_registry 看 method 是否 self-having
        let has_self = self.trait_registry.get(tname_str)
            .and_then(|info| info.methods.iter().find(|m| m.name == method))
            .map(|m| m.has_self)
            .unwrap_or(true);
        // v0.09: 找泛型化的具体 impl
        let impl_name = impl_method_key(tname_str, &trait_generics, &for_type, &for_generics, method);
        let env = self.environment.lock().unwrap();
        if let Some(task) = env.get(&impl_name) {
            drop(env);
            let mut all_args = if has_self { vec![receiver.clone()] } else { Vec::new() };
            all_args.extend(args);
            return self.call_value(&task, all_args);
        }
        drop(env);
        // v0.09: fallback 到默认实现（也带泛型）
        let default_name = default_impl_method_key(tname_str, &trait_generics, method);
        let env = self.environment.lock().unwrap();
        if let Some(task) = env.get(&default_name) {
            drop(env);
            let mut all_args = if has_self { vec![receiver.clone()] } else { Vec::new() };
            all_args.extend(args);
            return self.call_value(&task, all_args);
        }
        drop(env);
    }
    Err(format!(
        "trait dispatch at line {}: no impl for type '{}' method '{}' (searched: {}, generics: trait<{}> for {})",
        call_site.line, for_type, method,
        search_chain.iter().map(|s| s.to_string()).collect::<Vec<_>>().join(" → "),
        trait_generics.join(","), for_generics.join(","),
    ))
}
```

### 1.9 `call_function` 拦截 `Trait::new` 改动

```rust
// v0.09: 拦截 `Trait<...>::new("ForType")` 构造调用
fn call_function(&mut self, name: &str, args: Vec<Value>, call_site: Span) -> Result<Value, String> {
    // 解析 name 如 "Container<number>::new"
    if name.ends_with("::new") {
        let qualified = &name[..name.len() - 5];  // 去掉 "::new"
        if let Some(lt) = qualified.find('<') {
            // 解析 "Container<number>" → trait_name="Container", trait_generics=["number"]
            let trait_name = qualified[..lt].to_string();
            let generics_str = &qualified[lt+1..qualified.len()-1];
            let trait_generics: Vec<String> = generics_str
                .split(',').map(|s| s.trim().to_string()).collect();
            
            if self.trait_registry.contains_key(&trait_name) {
                let type_arg = args.get(0).map(|v| v.to_string()).unwrap_or_default();
                return self.construct_trait_instance(
                    &trait_name, &trait_generics, &type_arg, &[], call_site
                );
            }
        } else {
            // 无泛型: 旧逻辑
            let trait_name = qualified;
            if self.trait_registry.contains_key(trait_name) {
                let type_arg = args.get(0).map(|v| v.to_string()).unwrap_or_default();
                return self.construct_trait_instance(
                    trait_name, &[], &type_arg, &[], call_site
                );
            }
        }
    }
    // ... 已有逻辑
}
```

## 5. 调试技巧

### 5.1 dispatch key 不匹配

**症状**：`no impl for type 'X' method 'Y'`

**调试步骤**：
1. 查 `impl_table` 是否记录了 `__impl_<Trait>_<TraitGen>_<ForType>_<ForGen>_<method>`
2. 对比 dispatcher 构造的 key 与 impl_table 中的 key
3. 重点查 generics 字符串是否一致（`"number"` vs `"Number"`）

**调试代码**（临时）：
```rust
// 在 execute(Stmt::ImplDef) 后
eprintln!("Registered: {}", impl_method_name);
```

### 5.2 where 子句检查

**症状**：`where bound 'X' is not a known trait`

**原因**：where 子句 bound 写错（或 trait 拼写错误）

**修复**：检查 trait 是否在 trait_registry 中

### 5.3 泛型 TraitObject 解析

**症状**：dispatch 返回 TraitObject 的 for_generics / trait_generics 为空

**原因**：eval_dyn_trait 或 construct_trait_instance 没正确传递 generics

**调试**：在 dispatch_trait_method 入口打印：
```rust
eprintln!("Dispatch: for_type={}, for_generics={:?}, trait_name={}, trait_generics={:?}",
    for_type, for_generics, trait_name, trait_generics);
```
