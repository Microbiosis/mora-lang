## 1. v0.10 修复清单

| # | Finding | 修复方式 |
|---|---------|---------|
| F1 | AI 调用失败无 retry | retry 循环（最多 3 次 + base 1000ms 指数退避 + ±20% jitter） |
| F2 | 网络错误立刻失败 | ureq::Error::Transport 自动 retry |

## 2. 关键代码改动（src/interpreter.rs）

### 2.1 3 个 helper 函数

```rust
/// v0.10: retry 次数配置（默认 3）
fn ai_retry_max() -> u32 { /* MORA_AI_RETRY_MAX env var */ }

/// v0.10: retry 基础间隔（默认 1000ms）
fn ai_retry_base_ms() -> u64 { /* MORA_AI_RETRY_BASE_MS env var */ }

/// v0.10: 错误是否可重试
///   - 网络错误（Transport）
///   - HTTP 429 rate limit
///   - HTTP 5xx server error
fn is_retryable_error(err: &str) -> bool

/// v0.10: retry sleep 时间（指数退避 + jitter）
///   attempt=0 → base
///   attempt=1 → base * 2 + jitter
///   attempt=2 → base * 4 + jitter
fn retry_sleep_ms(attempt: u32, base_ms: u64) -> u64
```

### 2.2 real_ai_chat_inner retry 循环

```rust
// v0.10: retry 循环（exponential backoff + jitter）
let max_retries = ai_retry_max();
let base_ms = ai_retry_base_ms();
for attempt in 0..=max_retries {
    if attempt > 0 {
        let sleep = retry_sleep_ms(attempt - 1, base_ms);
        std::thread::sleep(Duration::from_millis(sleep));
    }
    match agent.post(&url) /* ... */ {
        // 成功：return
        // 错误：if is_retryable_error && attempt < max_retries → continue
        //       else → return Err
    }
}
```

## 4. 新增 8 条 retry 单测

`is_retryable_error`：
- `test_is_retryable_error_transport` — 网络错误可重试
- `test_is_retryable_error_http_429` — 429 rate limit 可重试
- `test_is_retryable_error_http_5xx` — 500/502/503/599 可重试
- `test_is_retryable_error_http_4xx_not_retryable` — 400/401/403/404 不可重试
- `test_is_retryable_error_other` — 其他错误不可重试

`retry_sleep_ms`：
- `test_retry_sleep_ms_exponential` — 指数退避验证
- `test_retry_sleep_ms_no_overflow` — 不溢出

`ai_retry_*`：
- `test_ai_retry_config_defaults` — 默认值不 panic

## 5. 配置（环境变量）

| 变量 | 默认 | 说明 |
|------|------|------|
| `MORA_AI_RETRY_MAX` | 3 | 最大重试次数（不计首次请求） |
| `MORA_AI_RETRY_BASE_MS` | 1000 | 首次重试前的等待基准（后续翻倍 + ±20% jitter） |

示例：
```bash
# 默认：4 次请求，最多 1+2+4 = 7 秒等待
./mora script.mora

# 禁用 retry（0 次）
MORA_AI_RETRY_MAX=0 ./mora script.mora

# 快速 retry（5 次，base 100ms）
MORA_AI_RETRY_MAX=5 MORA_AI_RETRY_BASE_MS=100 ./mora script.mora
```

## 6. 行为变化

### 之前（v0.09）
- 网络瞬断 → AI 调用立即失败，mora script 终止
- 服务器 500 错误 → 立即失败
- 429 rate limit → 立即失败（用户需手动 retry）

### 之后（v0.10）
- 网络瞬断 → 自动 retry 3 次（指数退避 1s/2s/4s + jitter），最坏情况 7+ 秒
- 服务器 500 → 自动 retry
- 429 rate limit → 自动 retry
- HTTP 4xx（除 429）→ 立即失败（client 错误 retry 无意义）
- 超过 retry 次数 → 返回最终错误（含 retry 链信息）
