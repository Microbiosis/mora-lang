## 1. 问题

**v0.10 之前**：`TcpListener::bind(&addr)?` 直接返回 io::Error，整个 server 启动失败。

实际场景：
1. **同 demo 重启**：前一个 server 进程未释放端口（TIME_WAIT 或 demo 阻塞）→ 新进程 `os error 10048`（Windows）或 `os error 98`（Linux）
2. **多个 demo 并发跑**：5 个 examples 全部用 4096 端口 → 前一个占着 → 后 4 个全部失败
3. **TIME_WAIT**：TCP 主动关闭后端口保留 30-120 秒（MSL），即使进程已退出，端口仍在 TIME_WAIT 状态

## 2. 修复清单

| # | Finding | 修复方式 |
|---|---------|---------|
| F1 | 端口硬绑失败直接 panic | 自动尝试下一个端口（4096, 4097, 4098, 4099） |
| F2 | TIME_WAIT 状态端口无法立即复用 | 设置 SO_REUSEADDR（跨平台 libc::setsockopt） |
| F3 | 跨平台差异（Windows/Linux 常量不同） | 硬编码常量：SOL_SOCKET=1 Unix / 0xFFFF Windows；SO_REUSEADDR=2/4 |
| F4 | 用户不知道实际端口 | 控制台 eprintln 提示切换（`requested port N unavailable, using M instead`） |
| F5 | README 没说明自动选端口 | 加"v0.11 起"段落 |

## 3. 关键代码改动（src/http_server.rs）

### 3.1 SO_REUSEADDR 跨平台设置

```rust
fn set_reuse_addr(listener: &TcpListener) {
    #[cfg(unix)]
    {
        use std::os::unix::io::AsRawFd;
        let fd = listener.as_raw_fd();
        let opt: libc::c_int = 1;  // enable
        let _ = unsafe {
            libc::setsockopt(fd, 1 /*SOL_SOCKET*/, 2 /*SO_REUSEADDR*/,
                &opt as *const _ as *const libc::c_void,
                std::mem::size_of_val(&opt) as libc::socklen_t)
        };
    }
    #[cfg(windows)]
    {
        use std::os::windows::io::AsRawSocket;
        let socket = listener.as_raw_socket();
        let opt: libc::c_int = 1;
        let _ = unsafe {
            libc::setsockopt(socket as libc::SOCKET, 0xFFFF /*SOL_SOCKET*/, 4 /*SO_REUSEADDR*/,
                &opt as *const _ as *const libc::c_char,
                std::mem::size_of_val(&opt) as libc::c_int)
        };
    }
}
```

### 3.2 自动选端口

```rust
fn bind_with_reuse(host: &str, requested_port: u16, max_attempts: u16) -> io::Result<(TcpListener, u16)> {
    for offset in 0..max_attempts {
        let port = match requested_port.checked_add(offset) {
            Some(p) => p,
            None => break,
        };
        match TcpListener::bind(format!("{}:{}", host, port)) {
            Ok(listener) => {
                set_reuse_addr(&listener);
                if offset > 0 {
                    eprintln!("[serve] requested port {} unavailable, using {} instead",
                        requested_port, port);
                }
                return Ok((listener, port));
            }
            Err(_) => continue,  // 端口被占，尝试下一个
        }
    }
    Err(io::Error::new(io::ErrorKind::AddrInUse,
        format!("could not bind to any port in range {}-{}",
            requested_port, requested_port.saturating_add(max_attempts - 1))))
}
```

### 3.3 start 函数集成

```rust
pub fn start(host: &str, port: u16, ...) -> io::Result<()> {
    // v0.11: 自动选端口（避免与之前未释放的进程冲突）
    let (listener, actual_port) = bind_with_reuse(host, port, 4)?;
    let addr = format!("{}:{}", host, actual_port);
    eprintln!("[serve] Mora HTTP server listening on http://{}", addr);
    // ... 后续不变
}
```

### 3.4 Cargo.toml 新增依赖

```toml
libc = { version = "0.2", features = ["extra_traits"] }
```

`extra_traits` feature 提供 SOL_SOCKET / SO_REUSEADDR 常量。硬编码数值作为 fallback（避免 feature 兼容性问题）。

## 5. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 依赖加 libc | libc 0.2 | 比 socket2 更轻量；ureq 已间接依赖 |
| SO_REUSEADDR 常量 | 硬编码 1/2/0xFFFF/4 | extra_traits feature 在不同 libc 版本不一致；硬编码跨平台稳 |
| max_attempts | 4 | 太多浪费；4 个端口足够避开绝大多数冲突 |
| 静默失败 | eprintln 提示 | SO_REUSEADDR 是 best-effort，不影响主流程 |
| 自动选 vs 失败 | 自动选 | 用户体验优先；生产环境可改回严格模式 |
